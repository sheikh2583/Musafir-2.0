  6.1   Prayer Tracking (Home Screen)

        - Persisted locally in AsyncStorage (@salat_tracking)

  6.2   Streak System
        - Consecutive-day streak tracking (up to 365 days)
        - Streak counted by iterating backwards day-by-day
        - All 5 prayers must be completed for streak day to count
        - Persisted in AsyncStorage (@salat_streak)

  6.3   Scoring Algorithm
        - Base: each prayer = 1 point
        - Consecutive multiplier:
          · Prayers in order: Fajr → Dhuhr → Asr → Maghrib → Isha
          · Each consecutive prayer multiplies: points × currentMultiplier, then multiplier++
          · Example: Fajr=1, Dhuhr=2, Asr=6, Maghrib=24, Isha=120 (factorial-like growth)
          · Isha → next-day Fajr counts as consecutive (overnight bridge)
        - Skipping a prayer resets multiplier to 1
        - Weekly reset: at 35 prayers (5×7 days), best score saved, counter resets
        - Best weekly score persisted for all-time leaderboard

  6.4   SalatLeaderboardScreen
        - Weekly / All-Time tab toggle
        - My Stats card:
          · Current rank
          · Weekly score
          · Streak multiplier
          · Prayers until weekly reset
        - Leaderboard list:

          · Avatar, name ("You" indicator for self)
          · Prayer count, streak multiplier, score
          · Gold/silver/bronze styling for top 3
          · Current user highlighted
        - Friends-only filtering (shows self + befriended users only)
        - "ما شاء الله" header with scale animation
        - Total participants count
        - Scoring Info button → ScoringInfo screen
        - Pull-to-refresh
        - Clears stale data before each fetch

  6.5   ScoringInfoScreen

        - Troy ounce → gram conversion: ÷ 31.1035

  23. ML/AI SEARCH PIPELINE
================================================================================

  23.1  Two-Stage RAG Search (Quran + Hadith)
        - Stage 1 — Embedding: BGE-base-en-v1.5 (768-dimensional vectors)
          · ONNX Runtime Node for inference
          · @xenova/transformers for tokenization
        - Stage 2 — Retrieval: ChromaDB HNSW index
          · Approximate nearest neighbor search
          · Returns top candidates
        - Stage 3 — Reranking: BGE Reranker (cross-encoder)
          · Cross-checks query-document relevance
          · Re-scores and sorts
        - Output: top 7 results in ~2-3 seconds

  23.2  ML Search Module Files (backend/ml-search/)
        - vector-search.js — Quran vector search engine
        - hadith-vector-search.js — Hadith vector search engine
        - ollama-search.js — Ollama-based Quran search
        - ollama-hadith-search.js — Ollama-based hadith search
        - query-normalizer.js — Query preprocessing and normalization

  23.3  Search Modes
        - QURAN_SEARCH_MODE env variable controls mode:
          · "vector" — use local BGE ONNX embeddings
          · Default — basic text search
        - Semantic search routes mounted BEFORE basic routes (priority)
